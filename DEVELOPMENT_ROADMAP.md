## 📅 Phase 0: 完全移植 Tuxemon（6周）

### Week 1-2: 核心引擎 + 资源加载

**目标：** 搭建 Web 游戏引擎基础框架

**任务清单：**
- [ ] Canvas 渲染引擎（游戏循环、帧动画）
- [ ] 资源加载器（图片、音频、JSON）
- [ ] 游戏状态机（标题→菜单→地图→战斗）
- [ ] 输入系统（键盘、手柄、触摸）
- [ ] 存档系统（LocalStorage）

**技术选择：**
- 纯 Canvas API（无框架）
- TypeScript 类型安全
- Web Audio API 音频

---

### Week 3-4: 地图系统 + NPC 系统

**目标：** 实现 TMX 地图解析和 NPC 交互

**任务清单：**
- [ ] TMX 地图解析器（tile layers、object groups）
- [ ] 瓦片渲染（支持多层、动画瓦片）
- [ ] 碰撞检测（墙壁、障碍物）
- [ ] 玩家移动（格子移动、平滑动画）
- [ ] NPC 系统（路径、对话、交互）
- [ ] 事件系统（触发器、条件判断）
- [ ] 场景切换（传送门、进出建筑）

---

### Week 5-6: 战斗系统完整移植

**目标：** 100% 还原 Tuxemon 战斗系统

**任务清单：**
- [ ] 战斗界面（回合制 UI）
- [ ] 怪物数据加载（411个怪物 JSON）
- [ ] 技能数据加载（274个技能 JSON）
- [ ] 属性克制系统（14种元素）
- [ ] 伤害计算公式
- [ ] 状态效果（中毒、灼烧、麻痹等35种）
- [ ] 战斗动画（技能特效）
- [ ] 捕捉系统
- [ ] 经验值和升级
- [ ] AI 对手（简单策略）

## 🛠️ 技术架构

```
前端（Web）
├── Canvas 渲染引擎
├── TypeScript
├── Web Audio API
└── WebSocket Client

后端（API）
├── Python FastAPI（游戏 API）
├── Node.js（WebSocket 实时通信）
├── MySQL（数据持久化）
└── Redis（缓存）

资源
├── Tuxemon 素材（GPL-3.0）
├── 411 怪物精灵
├── 274 技能数据
└── 50+ 地图文件
```
